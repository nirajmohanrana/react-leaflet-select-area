# API Reference

The package exports one component, one hook, and associated TypeScript types:

<CodeBlock language="ts">
{`import {
  SelectArea,
  useSelectArea,
  type SelectAreaBounds,
  type SelectAreaController,
  type SelectAreaTrigger,
} from "react-leaflet-select-area";`}
</CodeBlock>

## `SelectArea`

`SelectArea` mounts inside a `MapContainer` and binds the selection workflow to the active Leaflet map.

### Props

- `onBoundsChange?: (bounds: SelectAreaBounds | null) => void`
- `keepRectangle?: boolean`
- `showControl?: boolean`
- `controlPosition?: ControlPosition`
- `options?: PathOptions`
- `controller?: SelectAreaController`

### Behavior

- Desktop users can hold `Ctrl` and mouse drag.
- Mobile and touch users can use the built-in one-shot control when `showControl` is enabled.
- Valid selections call `onBoundsChange`, and clearing the persisted selection calls `onBoundsChange(null)`.
- Tap-only interactions do not emit bounds.

## `useSelectArea`

Use the hook when your app needs to trigger selection mode from outside the map.

<CodeBlock language="ts">
{`const controller = useSelectArea();

controller.enterSelectionMode();
controller.exitSelectionMode();
controller.toggleSelectionMode();
controller.clearSelection();`}
</CodeBlock>

### Controller State

- `startPoint`
- `endPoint`
- `rectangleBounds`
- `isDrawing`
- `isSelectionMode`
- `isShortcutPressed`

### Controller Actions

- `beginSelection`
- `updateSelection`
- `completeSelection`
- `cancelSelection`
- `clearSelection`
- `enterSelectionMode`
- `exitSelectionMode`
- `toggleSelectionMode`
- `setShortcutPressed`

The controller object can be passed into `SelectArea` through the `controller` prop to synchronize internal map behavior with your own UI.
